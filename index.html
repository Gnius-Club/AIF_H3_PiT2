📝 De un Vistazo: Comando Central

Orquestación y Auditoría: Conectando Frontends con Backends Automatizados.

🎯 Objetivo de Aprendizaje

Los estudiantes podrán: Conectar una interfaz de usuario con backends en Make.com o Python mediante Fetch API, utilizando técnicas de Vibe Coding para la implementación y auditorías de IA para garantizar la calidad del software.

🤖 Prompt de Referencia para XOLO (Solo para Maestro)

⚠️ NOTA: XOLO actúa como Senior Code Reviewer. Formato RCTFA: 
**Role**: Senior Fullstack Developer & Security Auditor.
**Context**: Estudiantes de preparatoria están integrando un frontend JavaScript con un backend de automatización (Make.com/Python).
**Task**: Revisa el código de integración Fetch proporcionado. Identifica 'Code Smells', riesgos de seguridad (como API keys expuestas) y errores de asincronía. 
**Format**: Lista estructurada de hallazgos y una versión refactorizada siguiendo principios de código limpio.
**Adjectives**: Crítico, constructivo y profesional.

⚡ Puntos Esenciales de la Misión

🏗️

Concepto Clave

Orquestación de APIs: Conectar el cliente con flujos lógicos externos (Webhooks/Agents).

🔧

Habilidad Técnica

Vibe Coding con Fetch: Generación y depuración de código asíncrono mediante prompts técnicos.

🚩

Dinámica: The Bug Bash

Sesión de auditoría intensiva para eliminar deuda técnica antes del despliegue.

🧠

Interacción con XOLO

XOLO actúa como el filtro de calidad profesional que valida la mantenibilidad del proyecto.

🚀 Actividad 1: Intro - El Puente de Datos

⏱️ 5:00

👨‍🏫 Rol del Maestro

  • Presentar el concepto de "Sistemas Desconectados" vs "Sistemas Orquestados".
  • Explicar cómo la Fetch API actúa como el tejido conector entre UI y Lógica.
  • Proyectar Diapositiva 1: La anatomía de una Request profesional.

👦 Rol del Alumno

  • Analizar su arquitectura actual e identificar los puntos de salida de datos (formularios, clics).

🧠 Actividad 2: Táctica 'Be The Algorithm' (Webhook Edition)

⏱️ 10:00

👨‍🏫 Rol del Maestro

  • Coordinar la simulación física: 1 Alumno es el Frontend (envía JSON), 1 es el Network (valida Headers), 1 es Make.com (procesa y responde).
  • Introducir variables de error: "¿Qué pasa si el servidor responde 500?".

👦 Rol del Alumno

  • Frontend: Redactar el objeto JSON de salida.
  • Make.com: Simular la lógica del escenario y devolver un JSON de confirmación o error.

📊 Diapositiva 2: El Ciclo de Vida del Dato

Request -> Webhook -> Processing -> Response

No solo es enviar datos, es esperar la promesa de que el backend ha hecho su trabajo.

💡 Actividad 3: Vibe Coding - Implementación Fetch

⏱️ 10:00

👨‍🏫 Rol del Maestro

  • Guiar la sesión de Vibe Coding: En lugar de teclear línea por línea, usar la IA para generar el boilerplate del Fetch.
  • Enfatizar el manejo de `async/await`.

🤖 Rol de XOLO

⏸️ XOLO EN ESPERA

👦 Rol del Alumno

  • Crear un prompt efectivo para generar la función de integración: "Genera una función asíncrona en JS para enviar un formulario a un webhook...".
  • Implementar y probar la comunicación con el backend (Make/Python).

📊 Diapositiva 3: Prompting para Integración

Dile a tu IA de desarrollo: "Crea una función asíncrona llamada 'sendData' que use Fetch API para enviar un objeto JSON a [URL]. Incluye manejo de errores con try/catch y valida que la respuesta sea ok."

🛡️ Actividad 4: Táctica 'The Bug Bash' con XOLO

⏱️ 10:00

👨‍🏫 Rol del Maestro

  • Introducir el concepto de "Auditoría de Seguridad": ¿Están las credenciales en el código? ¿El código es legible para un humano?
  • Supervisar la interacción con Xolo como revisor senior.

🤖 Rol de XOLO

✅ SENIOR CODE REVIEWER ACTIVADO

  • Detecta funciones "espagueti" y variables mal nombradas.
  • Valida que el manejo de errores sea robusto.
  • Refactoriza para cumplir estándares profesionales.

👦 Rol del Alumno

  • Entregar su código a Xolo con el prompt RCTFA de la Actividad 0.
  • Aplicar las correcciones sugeridas y documentar el cambio más importante.

⚡ Actividad 5: Cierre - Deuda Técnica

⏱️ 5:00

👨‍🏫 Rol del Maestro

  • Reflexionar sobre la importancia de la calidad sobre la cantidad de código.
  • Explicar que el "Vibe Coding" requiere supervisión constante de la IA.

👦 Rol del Alumno

  • Subir a Gnius Space: 1) Captura del código integrado y 2) Captura del prompt de auditoría usado en Xolo o IA externa.
  • Anotar una mejora pendiente para la próxima sesión (Deuda Técnica).
